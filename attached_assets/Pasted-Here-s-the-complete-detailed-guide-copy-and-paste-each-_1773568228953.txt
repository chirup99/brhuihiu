Here's the complete detailed guide — copy and paste each command exactly:

Before you start
Open your project folder in VS Code, then open the terminal:
Terminal → New Terminal (or press Ctrl + `` on Windows / Cmd + `` on Mac)

Step 1 — Install dependencies
Paste this and press Enter:

npm install
Wait until you see something like added 600 packages. This only needs to be done once.

Step 2 — Build the project
Paste this and press Enter:

npm run build
Wait until you see Done in Xms. This compiles your latest code into a dist/ folder.

Step 3 — Create the staging folder
Paste this whole block at once and press Enter:

mkdir -p aws-eb-staging/.ebextensions
Step 4 — Create the config files
Paste each block one at a time and press Enter after each:

Port config:

cat > aws-eb-staging/.ebextensions/port.config << 'EOF'
option_settings:
  aws:elasticbeanstalk:application:environment:
    PORT: 8081
  aws:elasticbeanstalk:container:nodejs:
    ProxyServer: nginx
EOF
Procfile:

echo "web: npm start" > aws-eb-staging/Procfile
.npmrc:

echo "production=true" > aws-eb-staging/.npmrc
.env — paste this and replace the placeholder values with your real credentials:

cat > aws-eb-staging/.env << 'EOF'
NODE_ENV=production
PORT=8081
AWS_REGION=ap-south-1
DYNAMODB_TABLE_NAME=Users
AWS_ACCESS_KEY_ID=your_real_key_here
AWS_SECRET_ACCESS_KEY=your_real_secret_here
LIVEKIT_API_KEY=your_real_livekit_key
LIVEKIT_API_SECRET=your_real_livekit_secret
EOF
Step 5 — Copy the required files
Paste this whole block at once and press Enter:

cp -r dist aws-eb-staging/ && cp package.json aws-eb-staging/ && cp package-lock.json aws-eb-staging/ && cp -r server aws-eb-staging/ && cp -r shared aws-eb-staging/ && cp tsconfig.json aws-eb-staging/
Step 6 — Create the ZIP with a custom name
Replace brs-connect-v1 with whatever name you want for your ZIP file:

On Mac or Linux:

cd aws-eb-staging && zip -r ../brs-connect-v1.zip . && cd ..
On Windows (PowerShell):

Compress-Archive -Path .\aws-eb-staging\* -DestinationPath brs-connect-v1.zip
Change brs-connect-v1 to anything — for example brs-connect-v2, my-app-march, release-15march — just keep the .zip at the end.

Step 7 — Clean up the staging folder
rm -rf aws-eb-staging
You're done!
Your ZIP file (e.g. brs-connect-v1.zip) will appear in your project folder — ready to upload to AWS Elastic Beanstalk.
