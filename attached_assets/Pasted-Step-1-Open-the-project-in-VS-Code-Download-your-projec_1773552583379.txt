Step 1 — Open the project in VS Code
Download your project files from Replit (use the three-dot menu → Download as ZIP), then open that folder in VS Code.

Step 2 — Open the Terminal in VS Code
Go to the top menu → Terminal → New Terminal. A terminal panel opens at the bottom.

Step 3 — Install dependencies
Type this and press Enter:

       npm install
Wait for it to finish (you'll see "added X packages").

Step 4 — Build the project
Type this and press Enter:

        npm run build
This creates a dist/ folder with the compiled app. Wait until you see Done in Xms.

Step 5 — Create the staging folder
        mkdir -p aws-eb-staging/.ebextensions
Step 6 — Create the config files
Port config — create a file at aws-eb-staging/.ebextensions/port.config:

            option_settings:
            aws:elasticbeanstalk:application:environment:
                PORT: 8081
            aws:elasticbeanstalk:container:nodejs:
                ProxyServer: nginx
Procfile — create aws-eb-staging/Procfile:

web: npm start
.env — create aws-eb-staging/.env with your real credentials:

            NODE_ENV=production
            PORT=8081
            AWS_REGION=ap-south-1
            DYNAMODB_TABLE_NAME=Users
            AWS_ACCESS_KEY_ID=your_real_key
            AWS_SECRET_ACCESS_KEY=your_real_secret
            LIVEKIT_API_KEY=your_real_key
            LIVEKIT_API_SECRET=your_real_secret
Step 7 — Copy the required files into staging
            cp -r dist aws-eb-staging/
            cp package.json aws-eb-staging/
            cp package-lock.json aws-eb-staging/
            cp -r server aws-eb-staging/
            cp -r shared aws-eb-staging/
            cp tsconfig.json aws-eb-staging/
Step 8 — Create the ZIP
On Mac/Linux:

cd aws-eb-staging && zip -r ../persona-eb-deployment.zip . && cd ..
On Windows (PowerShell):

            Compress-Archive -Path aws-eb-staging\* -DestinationPath persona-eb-deployment.zip
Step 9 — Clean up
                rm -rf aws-eb-staging
Done!
You'll now have persona-eb-deployment.zip in your project root — ready to upload to AWS Elastic Beanstalk via the AWS Console or EB CLI.

Tip: Every time you make code changes, just repeat Steps 4 and 7–9 to get a fresh ZIP.
